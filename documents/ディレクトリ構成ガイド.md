## `src/routes/` — ページ

Next.js の `app/` と同じ。**URLと1対1対応**します。

```tsx
// src/routes/index.tsx → localhost:5173/
export const Route = createFileRoute('/')({
  component: IndexPage,
});

function IndexPage() {
  return <MapView />; // features/map/ のコンポーネントを呼ぶだけ
}
```

ルートファイルは**薄く**保ちます。実際の中身は `features/` に書きます。

## `src/features/` — 機能の実装

**`features/map/components/MapView.tsx`**
```tsx
// MAP-01〜03 の実装
export function MapView() {
  const { spots } = useSpots(); // hooks/ からデータを取得

  return (
    <MapContainer>
      {spots.map(spot => <SpotMarker key={spot.id} spot={spot} />)}
    </MapContainer>
  );
}
```

**`features/map/hooks/useSpots.ts`**
```tsx
// Supabase からデータを取ってくる
export function useSpots() {
  const [spots, setSpots] = useState<Spot[]>([]);

  useEffect(() => {
    supabase.from('spots').select('*').then(({ data }) => {
      setSpots(data ?? []);
    });
  }, []);

  return { spots };
}
```

**ルールは1つ：`routes/` から `features/` を呼ぶ。`features/` 同士は直接呼ばない。**

## `src/hooks/useGeolocation.ts` — 複数featureで使うhook

```tsx
// MAP-05 でも LOC-05 でも同じ処理が必要なので共通化
export function useGeolocation() {
  const [position, setPosition] = useState(null);

  useEffect(() => {
    navigator.geolocation.getCurrentPosition(pos => {
      setPosition({ lat: pos.coords.latitude, lng: pos.coords.longitude });
    });
  }, []);

  return { position };
}
```

## `supabase/functions/post-comment/` — Edge Function（LOC-01〜04）

Supabase のサーバーサイド処理。位置検証はクライアント側ではなくここで行います。

```ts
// フロント側
await supabase.functions.invoke('post-comment', {
  body: { spot_id, body, user_lat, user_lng },
});

// サーバー側（Deno）
// → 位置を検証 → 範囲外なら403 → 範囲内ならDBにINSERT
```

## `supabase/migrations/` — DBの変更履歴（DB-01〜10）

```sql
-- 001_create_spots.sql
create table spots (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  lat float not null,
  lng float not null
);
```

SQLをファイルで管理することでチーム全員が同じDB構造を再現できます。
