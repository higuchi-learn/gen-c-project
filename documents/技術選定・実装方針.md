# 技術選定・実装方針

## 採用スタック

| レイヤー | 技術 | 採用理由 |
|---------|------|---------|
| フロントエンド | Vite + React (SPA) | SSR不要・Leafletのwindow問題を回避・MVP最速 |
| 地図 | Leaflet (react-leaflet) | 無料・カスタマイズ自由・クラスタリング対応 |
| BaaS | Supabase | PostGIS/Realtime/Auth/Storageが一体・位置情報クエリに最適 |
| サーバーレス | Supabase Edge Functions | 位置検証等のビジネスロジックをサーバーサイドで処理 |
| 言語 | JavaScript (JSDoc補完) | スピード優先。Supabaseクエリの多い箇所はJSDocで型補完 |

---

## Next.js を採用しない理由

- このサービスは**現地で地図を見ながら使う**ことが前提 → PWA的なSPA体験で十分
- LeafletはSSR環境（Node.js）で`window`を参照してクラッシュする
  - Next.jsでは `dynamic(() => import(...), { ssr: false })` で回避できるが、地図はどうせCSRでしか動かせないため、SSRの恩恵がない
- SEOが重要なのはスポット詳細ページのみ → MVPフェーズでは後回し可

**スケールしたら:** SEO需要が出てきた段階でNext.jsへの移行、またはSupabase Edge FunctionsでOGPのみ対応する。

## アーキテクチャ全体像

```
React (Vite SPA)
│
├── supabase-js（直接操作）
│     ├── スポット一覧・詳細取得        → PostgREST
│     ├── スポット登録                  → PostgREST + Storage（画像）
│     ├── 認証                         → Supabase Auth（Google OAuth等）
│     ├── 画像アップロード              → Supabase Storage
│     └── 新着スポット通知             → Supabase Realtime
│
└── supabase.functions.invoke（Edge Functions経由）
      ├── コメント投稿                  → 位置検証（PostGIS）→ INSERT
      └── 発見記録 + アチーブメント判定 → 最速到着チェック → 付与
```

## supabase-js の基本

```js
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  'https://xxxx.supabase.co',
  'your-anon-key'
)

// データ取得
const { data, error } = await supabase.from('spots').select('*')

// データ挿入
await supabase.from('spots').insert({ name: '謎の岩', lat: 35.01, lng: 135.77 })

// 認証
await supabase.auth.signInWithOAuth({ provider: 'google' })

// リアルタイム購読（新着スポット通知）
supabase
  .channel('spots')
  .on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'spots' }, (payload) => {
    console.log('新しいスポット:', payload.new)
  })
  .subscribe()
```

## Edge Functions の基本

Supabase が提供するサーバーレス関数。Deno ランタイム上で動作。

```js
// supabase/functions/post-comment/index.js
import { createClient } from '@supabase/supabase-js'

Deno.serve(async (req) => {
  const { spot_id, body, user_lat, user_lng } = await req.json()

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL'),
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')
  )

  // PostGIS で距離判定（サーバーサイド = 偽装不可）
  const { data } = await supabase.rpc('is_within_range', {
    spot_id, user_lat, user_lng, range_km: 0.5
  })

  if (!data?.within_range) {
    return new Response(JSON.stringify({ error: '現地限定コメントです' }), { status: 403 })
  }

  const { data: comment, error } = await supabase.from('comments').insert({ spot_id, body })
  return new Response(JSON.stringify({ comment, error }))
})
```

### クライアントからの呼び出し

```js
const { data, error } = await supabase.functions.invoke('post-comment', {
  body: { spot_id: 'xxx', body: 'すごい岩', user_lat: 35.01, user_lng: 135.77 }

```

## 処理の分担：直接 vs Edge Functions

| 処理 | 方法 | 理由 |
|------|------|------|
| スポット一覧・詳細 | supabase-js 直接 | 単純な読み取り |
| スポット登録 | supabase-js 直接 | RLSで自分のデータのみ書けるよう制御 |
| 画像アップロード | supabase-js 直接 | Storage APIで完結 |
| 新着通知 | supabase-js 直接 | Realtimeで購読 |
| **コメント投稿** | **Edge Functions** | **位置検証が必要。クライアント任せにしてはいけない** |
| **発見記録・アチーブメント** | **Edge Functions** | **複数テーブルへのトランザクション + 判定ロジック** |
| ランキング | supabase-js + DB View/RPC | 集計はSQL関数としてDB側に持つ |

## RLS（Row Level Security）設計方針

supabase-js からのリクエストはRLSで制御する。

```sql
-- スポットは誰でも読める
create policy "全員読み取り可" on spots for select using (true);

-- スポット登録は本人のみ
create policy "本人のみ登録可" on spots for insert
  with check (auth.uid() = user_id);

-- コメントの読み取りはEdge Functions経由のみ（service_roleキー使用）
-- → RLSでは位置情報を検証できないため、直接読み取りを禁止し
--   Edge Functions内でサービスロールキーを使って制御する
```

## 位置情報（PostGIS）のSQL関数

```sql
-- PostGIS拡張を有効化
create extension if not exists postgis;

-- spotsテーブルにgeography型カラムを追加
alter table spots add column location geography(Point, 4326);

-- 距離判定関数
create or replace function is_within_range(
  spot_id uuid, user_lat float, user_lng float, range_km float
)
returns table(within_range boolean) as $$
  select ST_DWithin(
    location::geography,
    ST_MakePoint(user_lng, user_lat)::geography,
    range_km * 1000
  ) as within_range
  from spots where id = spot_id
$$ language sql;
```
