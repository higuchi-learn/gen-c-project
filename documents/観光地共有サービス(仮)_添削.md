# 観光地共有サービス(仮) — 機能添削・優先度整理

## ミッション再確認

| 軸         | 内容                                                                                               |
| ---------- | -------------------------------------------------------------------------------------------------- |
| ミッション | 旅行・観光地をより面白く / 国内観光消費を増やす・オーバーツーリズム解消 / 新たな観光スポットの創出 |
| ビジョン   | 観光地での時間を充実させる / 旅行に新しい価値を提供（小さな発見の喜び）/ 場所への理解を深める      |

> **コアバリュー（整理）:**
> 「わざわざ紹介するほどでもない発見」を共有し、その場に実際に行くことで初めて体験できる価値を提供する。

---

## 機能ごとの評価と優先度

優先度は **Must / Should / Nice / ✗ 削除推奨** の4段階。

---

### ✅ Must（MVP必須）

#### 1. ミニスポット登録

**整合性: ◎ 最重要**

- 位置・画像・説明を登録できる → そのまま維持
- これまで自分が登録したスポットを閲覧できる → 維持

> プロダクトの根幹。これなしには何も始まらない。

---

#### 2. 地図上でのミニスポット閲覧（★追加推奨）

**整合性: ◎ 最重要**

> 現状の草案に明示されていないが、**マップ UI でスポットを探せること**はコア体験の前提。必ず明記・実装すること。

---

#### 3. コンテンツ公開レベルの設計（Location Gate → ティーザー方式に変更）

**整合性: ◎**

事前のモチベーション形成と現地体験の驚きを両立するため、情報種別ごとに公開範囲を分ける。

| 情報       | 公開範囲                     | 理由                                                                           |
| ---------- | ---------------------------- | ------------------------------------------------------------------------------ |
| 位置・説明 | 常時公開                     | 旅行前の動機形成に必要。どんなスポットか分からないと使う気にならない           |
| 画像       | タップで任意表示（常時）     | 画像がないと価値が伝わらないスポットもある。見るかどうかは利用者の自由         |
| コメント   | **現地限定**（半径n km以内） | 「実際に来た人の生の声」を現地でのみ解放することで、訪問動機と発見の喜びを保持 |

> **設計意図:**
>
> - 説明・画像で「行きたい」を事前に作り、コメントで「来て良かった」を現地で提供する
> - 画像をあえて見ないユーザーには現地での発見体験が残る（ユーザーの自由に委ねる）
> - コメントの現地限定により、ビジョン「発見する喜び」の一部を能動的に保護できる

---

### ✅ Should（早期実装）

#### 4. コメント機能

**整合性: ○**

- コメント自体 → 維持（場所への理解を深める、喜びの共有）
- **いいね数で文字を大きく** → 維持。視覚的に「現地の声の重み」が伝わり面白い

---

#### 5. ランキング機能（発見数・登録数）

**整合性: ○**

- 発見した数のランキング → **維持**（ミニスポット発見を目的にさせる動機付け）
- 登録数のランキング → 維持（地元民の投稿を促進 → 新スポット創出に直結）
- **季節別ランキング** → Should の下位。例（花粉・日陰）はミニスポットらしくて面白いが、実装コストと相談

---

#### 6. 旅行記録

**整合性: ○**

- 訪れた場所・いいねした場所を振り返れる → 維持
- 「旅行の新しい価値」を手元に残す体験として有効

---

#### 7. 新着ミニスポット通知 ＋ 最速到着アチーブメント

**整合性: ○**

- 通知 → 維持
- **最速到着アチーブメント → 強く維持**。「実際にその場に行く」行動を直接促進し、ミッションと完全に一致する

---

### 🔵 Nice（余裕があれば）

#### 8. コメントが薄れて消える（Decay機能）

**整合性: △**

- 誰かが見る・反応すると時間延長 → アイデアとしては面白い
- ただし**コンテンツ消失への不満リスク**と実装複雑度を考えると後回しでよい
- ミッションへの貢献は間接的

---

#### 9. タイムテーブル機能

**整合性: △**

- 場所と滞在時間の記録 → 旅の記録として有用
- 平均滞在時間の統計 → 他の旅行者の参考になる
- ただしコア体験ではないため後回し

---

#### 10. 登録者・閲覧者の出身地統計

**整合性: △**

- どこから来たのか / 次に行った場所 → 「現地の人との話題作り」や観光消費データとして面白い
- ⚠️ **プライバシー設計が必要**。個人が特定されない集計表示であることを明示すること
- MVPには不要

---

### ✗ 削除推奨

#### ❌ リアクション機能（オリジナルステッカー）

**整合性: ✗ 薄い**

> 画像を切り取りステッカーを貼る機能は、SNS的な楽しさはあるが**ミニスポット発見・共有・現地訪問の促進**というミッションとの関連が薄い。
> 開発コストに対してコア体験への貢献が小さいため削除を推奨。

---

#### ❌ 季節限定コメント

**整合性: ✗**

> 季節別ランキングで季節性は十分に表現できる。コメント単位で季節を管理するのは複雑さの割に効果が薄い。

---

## 優先度サマリー

| 優先度     | 機能                                       | 理由                                          |
| ---------- | ------------------------------------------ | --------------------------------------------- |
| **Must**   | ミニスポット登録                           | コア                                          |
| **Must**   | マップでのスポット閲覧（★追加）            | コア体験の前提                                |
| **Must**   | コンテンツ公開レベル設計（ティーザー方式） | 旅前の動機形成と現地体験の両立                |
| **Should** | コメント（いいね付き）                     | 共有・理解の深化                              |
| **Should** | 発見数・登録数ランキング                   | 行動促進・投稿促進                            |
| **Should** | 旅行記録                                   | 旅の新価値                                    |
| **Should** | 新着通知 ＋ 最速アチーブメント             | 現地訪問を促進                                |
| **Nice**   | Decayコメント                              | 面白いが後回し                                |
| **Nice**   | タイムテーブル                             | 旅の記録として有用                            |
| **Nice**   | 出身地統計                                 | データとして面白いがMVP外・要プライバシー設計 |
| **Nice**   | 季節別ランキング                           | ランキング実装後に追加                        |
| **✗ 削除** | オリジナルステッカー                       | ミッション整合性が薄い・開発コスト大          |
| **✗ 削除** | 季節限定コメント                           | 季節別ランキングで代替可能                    |

---

## 追加で検討すべきこと

1. **ミニスポットのカテゴリ/タグ** — 「絶景」「休憩」「穴場」など。マップ上での絞り込み検索に使える
2. **オフラインキャッシュ** — 電波が弱い観光地での使用を考えると、エリアを事前DLできると体験が向上する
3. **投稿の品質担保** — 不適切コンテンツへの報告機能はリリース前に最低限必要
