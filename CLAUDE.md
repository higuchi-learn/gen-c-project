# CLAUDE.md — 観光地共有サービス(仮) プロジェクト指針

## プロジェクト概要

「わざわざ紹介するほどでもない発見（ミニスポット）」を共有し、旅行の新しい価値を提供するサービス。

| 軸 | 内容 |
|----|------|
| ミッション | 旅行・観光地をより面白く / 国内観光消費を増やす・オーバーツーリズム解消 / 新たな観光スポットの創出 |
| ビジョン | 観光地での時間を充実させる / 旅行に新しい価値を提供（小さな発見の喜び）/ 場所への理解を深める |

---

## 仕様変更・機能追加時のチェックルール

**仕様の定義・変更を行うたびに、必ず以下を確認すること。**

1. **ミッションとの整合性** — その機能は「旅行を面白くする」「観光消費を増やす」「新スポットを創出する」のいずれかに貢献しているか
2. **ビジョンとの整合性** — 「発見する喜び」「旅の新しい価値」「場所への深い理解」を損なっていないか
3. **コアバリューとの整合性** — 「実際にその場に行く」動機・体験を阻害していないか
4. **トレードオフの明示** — 整合性に緊張が生じる場合は、その理由と許容できる理由を明記する

## 型・データアクセス

- DBのテーブル型は `src/types/database.types.ts` に自動生成済み（Supabase CLIで生成）
- 各テーブルの型エイリアスは `src/types/index.ts` に定義済み
- コードを書く際は必ず `src/types/index.ts` の定義済み型を使用すること（独自に型を定義しない）
- Supabaseクライアントは `src/lib/supabase.ts` からインポートする
- CRUD操作の例は `documents/supabase-crud-guide.md` を参照

| 型名 | 対応テーブル | 用途 |
|------|-------------|------|
| `Spot` / `SpotInsert` / `SpotUpdate` | `spots` | スポット |
| `Comment` / `CommentInsert` / `CommentUpdate` | `comments` | コメント |
| `Discovery` / `DiscoveryInsert` | `discoveries` | 発見記録 |
| `Achievement` / `AchievementInsert` | `achievements` | アチーブメント |

## 対話の進め方

- 機能の追加・変更提案は `_添削.md` に反映する
- 仕様が固まるたびに優先度サマリーテーブルを最新状態に保つ
- ユーザーの発言から仕様変更の意図を読み取ったときは、**ミッション・ビジョンとの整合性への影響を先に指摘してから** ドキュメントに反映する
- 機能削除・優先度変更を提案するときは理由を簡潔に添える

## 現在確定している主要仕様

### コンテンツ公開レベル（ティーザー方式）

| 情報 | 公開範囲 |
|------|----------|
| 位置・説明 | 常時公開 |
| 画像 | タップで任意表示（常時・ユーザーの自由） |
| コメント | 現地限定（半径n km以内） |

### 機能優先度

| 優先度 | 機能 |
|--------|------|
| Must | ミニスポット登録、マップでのスポット閲覧、コンテンツ公開レベル設計 |
| Should | コメント（いいね付き）、発見数・登録数ランキング、旅行記録、新着通知＋最速アチーブメント |
| Nice | Decayコメント、タイムテーブル、出身地統計、季節別ランキング |
| 削除 | オリジナルステッカー、季節限定コメント |
